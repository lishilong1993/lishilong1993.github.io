概述

简单介绍Markdown

Markdown 是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式编写文档，然后转换成格式丰富的HTML页面。 —— 来自[维基百科]

简单的来说，Markdown就是用“标记符号”表示“格式”。Markdown语法标签与HTML语法标签是一一对应的，比如Markdown的二级标题标签##就对应着HTML中的h2标记，而且Markdown是兼容HTML语法的，如果你比较喜欢 HTML 的a 或 img 标签，可以直接在Markdown文档中使用这些标记，而不用 Markdown 提供的链接或是图像标签语法。

CSDN Markdown的功能支持

CSDN Markdown编辑器是使用StackEdit修改而来的，但在StackEdit的基础上做了一些细节优化，使得体验更加友好，CSDN Markdown支持的功能很多：

  • 实时效果预览并自动滚动定位到编辑处（所见即所得）
  • 支持 Markdown Extra / GitHub Flavored Markdown 扩展语法
  • 支持代码语法高亮
  • 支持 LaTeX 数学公式（基于 MathJax）
  • 支持UML序列图和流程图
  • 在线或离线管理多份 MarkDown 文档
  • 支持离线编辑
  • 支持导入在线和本地Markdown文档进行编辑
  • 可以将编辑内容导出为 Markdown，HTML文件
  • 管理已发布的文档

在这篇文章，我先介绍CSDN Markdown的基本语法，我会从Markdown语法说明、例子、与其对应的HTML语法、例子效果、注意事项这几个方面进行详细介绍。后面会继续介绍如何插入图片、CSDN Markdown的扩展语法、如何写数学公式和画图等相关的内容。

标题

Markdown 支持两种形式的标题语法：Setext形式和atx形式。

Setext 形式

Markdown语法说明：

Setext 形式是采用底线的方式，利用若干个 = （最高阶标题）和 - （第二阶标题）完成。

例子：

这是标题1

=============

这是标题2

–

注意事项：

这里#或-的数目是任意的，既可以写一个也可以写多个。

atx形式

Markdown语法说明：

atx 形式则是在行首插入 1 到 6 个 # ，分别对应到标题 1 到 6 阶。

例子：

#这是标题1

## 这是标题2

### 这是标题3

#### 这是标题4

##### 这是标题5

###### 这是标题6

转化为HTML后分别对应<h1>...<h6>这6个标记。

分隔线

Markdown语法说明：

你可以在一行中用三个或以上的星号*、减号-、底线_来建立一个分隔线，行内不能有其他东西。你也可以在星号或是减号中间插入空格。下面每种写法都可以建立一条分隔线。

例子：

* * *

***

*****

- - -

---------------------------------------

<hr />

Markdown的链接可分为自动链接和普通文本链接两种类型。

自动链接（可点击的URL和Email）

Markdown支持以比较简短的自动链接形式来处理网址和电子邮件信箱。

Markdown语法说明：

对于本身是个URL和Email的文本来说，只要是用尖括号包起来，Markdown就会自动把它转成链接。

例子：

这是一个可点击的URL：
http://www.csdn.net/

这是一个可点击的Email：
address@example.com

JAVA常用算法手册之阅读笔记<一>

数据结构：

数据：信息的载体；整数、字符、字符串、图像、声音

数据元素：数据的基本单位，一个数据元素由若干个数据项组成；

数据项：具有独立含义的最小标识单位，可称为：字段、域、属性

数据结构：数据之间的相互关系，数据的组织形式

内容：

1、数据的逻辑结构

2、数据的存储结构

3、数据的运算

数据结构分类：

1、线性结构

2、非线性结构

存储方式：

1、顺序存储

2、链接存储

3、索引存储

4、散列存储

数据类型：

1、基本数据类型

2、聚合数据类型

ADT:抽象数据类型

数据抽象、数据封装

常用数据结构：

1、数组（Array）

2、栈（Stack）:特殊的线性表；按照后进先出的原则存储数据；没有数据的栈为空栈；

3、队列（Queue）:特殊的线性表；按照先进先出的原则存储数据；没有数据的队列为空队列；

4、链表（Linked List）:一种数据元素按照链式存储结构进行存储的数据结构，特点为非连续；链表由系列数据结点构成，每个数据结点包括数据域和引用域；引用域保存数据结构下一个元素的存储地址。

5、树（Tree）:非线性结构；只有一个根节点

6、图（Graph）:非线性结构；数据节点成为顶点，边是顶点的有序偶对；顶点间有边表示两个顶点具有相邻关系。

7、堆（Heap）:树型据结构，一般讨论二叉堆；特点：根节点的值是所有节点中最小或最大的，根节点的两个子树也是堆结构

8、散列表（Hash）:如果在结构中存在关键字和T相等的记录，必定在F(T)的存储位置可以找到该记录。可以不用进行比较直接取得记录。

顺序表操作实例代码

package cn.sunline.oa.test;

import java.util.Scanner;

public class test02List {

public static void main(String[] args) {
    SLType SL=new SLType();

    System.out.println("順序表操作演示：");
    SL.SLInit(SL);
    System.out.println("初始化完成。。");

    Scanner sc=new Scanner(System.in);

    do{
        System.out.println("输入添加的节点信息（学号 姓名 年龄）：");
        DATA data=new DATA();
        data.key=sc.next();
        data.name=sc.next();
        data.age=sc.nextInt();

        if(data.age!=0){
            if(SL.SLAdd(SL, data)==0){
                break;
            }
        }else{
            break;
        }

    }while(true);
    System.out.println("顺序表中的节点顺序为：");
    SL.SLAll(SL);


    System.out.println("要取出节点的序号：");
    int i=sc.nextInt();

    DATA pdata=SL.SLFindByNum(SL, i);
    if(pdata!=null){
        System.out.println("第"+i+"个节点为："+pdata.key+","+pdata.name+","+pdata.age);
    }


    System.out.println("要查找的节点关键字：");
    String key=sc.next();
    int j=SL.SLFindByCont(SL, key);
    DATA pdata1=SL.SLFindByNum(SL, j);
    if(pdata1!=null){
        System.out.println("关键字"+key+"为第"+j+"个节点，该节点为："+pdata1.key+","+pdata1.name+","+pdata1.age);
    }

    System.out.println("插入节点(节点位置，学号，姓名，年龄)：");
    int l=sc.nextInt();
    DATA data2=new DATA();
    data2.key=sc.next();
    data2.name=sc.next();
    data2.age=sc.nextInt();
    SL.SLInsert(SL, l, data2);

    System.out.println("所有节点：");
    SL.SLAll(SL);
}
}

class DATA{ //节点关键字
 String key;
 String name;
 int age;
}

class SLType{ //定义顺序表结构
static final int MAXLEN=100;
DATA[] ListData=new DATA[MAXLEN+1];//保存顺序表的结构数组
int ListLen;            //顺序表已存在节点的数量

void SLInit(SLType SL){//初始化顺序表
    SL.ListLen=0;      //初始化为空表
}

int SLLength(SLType SL){
    return SL.ListLen;   //返回顺序表的元素数量
}

//插入节点
int SLInsert(SLType SL,int n,DATA data){
    int i;

    if(SL.ListLen>=MAXLEN){ //顺序表节点数量已超过最大数量
         System.out.println("顺序表已满，不能插入节点");
         return 0;
    }
    if(n<1||n>SL.ListLen-1){
        System.out.println("插入元素序号错误，不能插入元素！");
        return 0;
    }
    for(i=SL.ListLen;i>=n;i--){ //将顺序表中的数据向后移动
        SL.ListData[i+1]=SL.ListData[i];
    }
    SL.ListData[n]=data; //插入节点
    SL.ListLen++;  //顺序表节点数量增加1
    return 1;
}

//追加节点
int SLAdd(SLType SL,DATA data){ //增加元素到顺序表尾部
    if(SL.ListLen>=MAXLEN){ 
        System.out.println("顺序表已满，不能再添加节点！");
        return 0;
    }
    SL.ListData[++SL.ListLen]=data;
    return 1;
}

//删除节点
int SLDelete(SLType SL,int n){

    if(n<1||n>SL.ListLen+1){ //删除节点的序号不正确
        System.out.println("删除节点序号错误，不能删除节点");
        return 0;
    }
    for(int i=n;i<SL.ListLen;i++){ //将顺序表中的数据向前移动
        SL.ListData[i]=SL.ListData[i+1];
    }
    SL.ListLen--;
    return 1;
}

//根据序号查找节点
DATA SLFindByNum(SLType SL,int n){
    if(n<1||n>SL.ListLen+1){
        System.out.println("节点序号错误，不能返回节点");
        return null;
    }
    return SL.ListData[n];

}

//根据关键字查找节点
int SLFindByCont(SLType SL,String key){
    for(int i=1;i<=SL.ListLen;i++){
        if(SL.ListData[i].key.compareTo(key)==0){
            return i;
        }
    }
    return 0;
}

//显示表中的所有节点
int SLAll(SLType SL){
    for(int i=1;i<=SL.ListLen;i++){
        System.out.println(SL.ListData[i].key+","+SL.ListData[i].name+","+SL.ListData[i].age);
    }
    return 0;
}
}

学习笔记

今日笔记

中间件产品：

  • 中间件产品本身的作用就是连接、过渡，为其所属物品实现更好的价值和功用。他可以是一种产品到另一种产品的过渡，像电脑集成显卡到独立显卡的过渡品，亦半集成半独立，也可以是一件产品的成为产品所需的一个的连接部件。这个连接部件的作用就由它被创造出来时赋予的功用来确定了。像我们每个人出身时父母都要起名字一样，成为产品时，已经赋予它责任、功用。

  • 介于客户机和服务器之间的夹层，突破了传统的c/s架构，为构建大规模，高性能，分布式c/s应用程序提供了通信，事物，安全，容错等基础服务，屏蔽了底层应用细节，应用程序不必从底层开发，以自身的复杂性换取应用程序开发的简单

Tuxedo(Transaction for Unix has been Extended for Distributed Operation):

分布式操作扩展之后的Unix事务系统
是一个客户机/服务器的“中间件”产品，它在客户机和服务器之间进行调节，以保证正确地处理事务;
是一个事务处理(TP)监督器，它管理联机事务处理(OLTP)系统操作的事务.

Radware硬件负载均衡：

  • Radware+Tuxedo多机组合模式:解决Radware硬件负载均衡需要间断时间的缺点；实现应用不间断运行。

应用层负载均衡：LBR软负载

  • 提供主服务器和备份服务器（可以多台），主服务器不可用时，才使用备份服务器。

对接模式：

  • ATMI:application-to-Transaction Monitor Interface;应用程序到事务监听器接口；调用Tuxedo ATMI函数接入Tuxedo服务。
  • WTC:WebLogic Tuxedo Connector；实现WebLogic容器接入Tuxedo服务。
  • JOLT:实现JAVA调用Tuxedo服务。

缓冲区数据类型：

客户端与服务器，服务器与服务器是由数据缓冲区（通讯报文）来传递数据。

* STRING:是以空值结尾的单字字符串
* CARRY:有长度定义的二进制数据
* VIEW:类似于C的structure
* FML:固定结构的自定义缓冲 <

  • JSON报文：

性能测试：

可扩展性、可靠性、可维护性、可用性、安全性、兼容性、容灾

批量优化操作：

  • 接入层——》调度层——》作业层———》数据层

  • 批量任务队列—>数据层(确保一个批量任务只能有一个调度JVM执行)—>调度层（调度JVM,调度批量任务）—>作业层（作业JVM，执行批量作业）

大数据批量优化：

  • 原则：

    1、水平扩展：通过水平扩展，提升app端、db端吞吐量；

    2、数据切分：（1）水平切分：分库、分表（2）垂直切分：按业务、产品、功能等维护切分；即按模块分库

    3、其他：减少数据库的交互次数；离线加工；

  • 优化方法：

1、作业JVM水平扩展 2、DB水平扩展 3、优化逻辑，减少DB交互次数 4、分库 5、分表 6、按功能分库 7、离线加工（将数据从主库中通过ETL工具抽取到大数据平台）
