今日笔记

中间件产品：

  • 中间件产品本身的作用就是连接、过渡，为其所属物品实现更好的价值和功用。他可以是一种产品到另一种产品的过渡，像电脑集成显卡到独立显卡的过渡品，亦半集成半独立，也可以是一件产品的成为产品所需的一个的连接部件。这个连接部件的作用就由它被创造出来时赋予的功用来确定了。像我们每个人出身时父母都要起名字一样，成为产品时，已经赋予它责任、功用。

  • 介于客户机和服务器之间的夹层，突破了传统的c/s架构，为构建大规模，高性能，分布式c/s应用程序提供了通信，事物，安全，容错等基础服务，屏蔽了底层应用细节，应用程序不必从底层开发，以自身的复杂性换取应用程序开发的简单

Tuxedo(Transaction for Unix has been Extended for Distributed Operation):

分布式操作扩展之后的Unix事务系统
是一个客户机/服务器的“中间件”产品，它在客户机和服务器之间进行调节，以保证正确地处理事务;
是一个事务处理(TP)监督器，它管理联机事务处理(OLTP)系统操作的事务.

Radware硬件负载均衡：

  • Radware+Tuxedo多机组合模式:解决Radware硬件负载均衡需要间断时间的缺点；实现应用不间断运行。

应用层负载均衡：LBR软负载

  • 提供主服务器和备份服务器（可以多台），主服务器不可用时，才使用备份服务器。

对接模式：

  • ATMI:application-to-Transaction Monitor Interface;应用程序到事务监听器接口；调用Tuxedo ATMI函数接入Tuxedo服务。
  • WTC:WebLogic Tuxedo Connector；实现WebLogic容器接入Tuxedo服务。
  • JOLT:实现JAVA调用Tuxedo服务。

缓冲区数据类型：

客户端与服务器，服务器与服务器是由数据缓冲区（通讯报文）来传递数据。

* STRING:是以空值结尾的单字字符串
* CARRY:有长度定义的二进制数据
* VIEW:类似于C的structure
* FML:固定结构的自定义缓冲 <

报文：

  • XML报文：报文头+XML报文体
  • JSON报文：

性能测试：

可扩展性、可靠性、可维护性、可用性、安全性、兼容性、容灾

批量优化操作：

  • 接入层——》调度层——》作业层———》数据层

  • 批量任务队列—>数据层(确保一个批量任务只能有一个调度JVM执行)—>调度层（调度JVM,调度批量任务）—>作业层（作业JVM，执行批量作业）

大数据批量优化：

  • 原则：

    1、水平扩展：通过水平扩展，提升app端、db端吞吐量；

    2、数据切分：（1）水平切分：分库、分表（2）垂直切分：按业务、产品、功能等维护切分；即按模块分库

    3、其他：减少数据库的交互次数；离线加工；

  • 优化方法：

1、作业JVM水平扩展 2、DB水平扩展 3、优化逻辑，减少DB交互次数 4、分库 5、分表 6、按功能分库 7、离线加工（将数据从主库中通过ETL工具抽取到大数据平台）
